================================================================================
                              WHAT IS PMO-SWARM?
================================================================================

PMO-Swarm is an autonomous multi-agent system that automates project management
office (PMO) tasks. It runs on a schedule (every 60 minutes) on Google Cloud Run
and produces daily Operating Briefs.

CAPABILITIES:
  1. Scans Jira for stalled tickets (>24h no activity)
  2. Audits teams for ownership gaps and RACI violations
  3. Checks for data hygiene (missing fields, stale status)
  4. Drafts escalation messages for critical blockers
  5. Synthesizes findings into daily Operating Brief

ARCHITECTURE:
  • 1 Orchestrator Agent (routes, gates, synthesizes)
  • 5 Skill Agents (execution_tracking, follow_up, ownership_raci,
    feature_completeness, hygiene)
  • All agents run in parallel via Google ADK
  • Governance gates enforce human approval before Jira writes

PERFORMANCE:
  ✓ Cycle time: Target <60 seconds (currently ~45 seconds)
  ✓ Jira API calls: Optimized JQL queries (batched)

  ✓ BigQuery: ACID guaranteed (append-only, upsert on conflict)
  ✓ AlloyDB: 30-day retention (auto-cleanup beyond 30 days)
  ✓ Trust Ledger: Append-only JSON lines (never edited)

DATA QUALITY:
  ✓ Governance gates: 100% coverage before Jira writes
  ✓ Operating Brief: Generated daily, no data loss
  ✓ Audit trail: Every decision logged to Trust Ledger
  ✓ Deduplication: BigQuery upsert prevents duplicate cycles

================================================================================
                            TROUBLESHOOTING
================================================================================

ISSUE: Container called exit(0) but no data in BigQuery
STATUS: ✅ Normal — AlloyDB and local files still have data
FIX: Check BigQuery permissions for pmo-agent-svc service account

ISSUE: Governance gates show "pending" but no one can approve
FIX: Wire gate approval flow to Slack/PagerDuty (see DEPLOY.md)

ISSUE: Jira queries timeout after 3600 seconds
FIX: Reduce SCAN_INTERVAL in config_registry, or optimize JQL queries

ISSUE: Cloud Run job fails to authenticate
FIX: Verify GOOGLE_APPLICATION_CREDENTIALS and service account permissions

See GETTING_STARTED.md (Troubleshooting) and DEPLOY.md (Common Issues) for more.
